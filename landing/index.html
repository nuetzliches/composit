How it works

Three commands. IS-state vs. SHOULD-state. No SaaS required.

1

Scan

Nine built-in scanners detect Docker, Terraform, Caddy, workflows, Prometheus, cron, env files, and MCP configs. Zero setup.

composit scan
2

Declare

Write a Compositfile in HCL. Approved providers, budgets, resource constraints, policy references.

vim Compositfile
3

Diff

Compare reality against intent. Get errors and warnings with severity and rule IDs. Terminal, JSON, YAML, or HTML output.

composit diff

Quick start

The CLI is free and open source (MIT). Point it at any repo — it reads; it never writes.

Install 
# Rust toolchain required. npx distribution is on the roadmap.
cargo install --git https://github.com/nuetzliches/composit
Scan any repo — IS-state 
# Writes composit-report.yaml with everything detected.
composit scan
Declare intent — SHOULD-state 
# Compositfile
workspace "my-stack" {
  resources {
    allow "docker_service" { max = 20 }
    require "workflow"      { min = 1  }
  }
}
Diff — catch drift 
# Terminal, JSON, YAML, or a standalone HTML report.
composit diff --output html

See the examples directory for a full Compositfile and matching report.

Which direction should composit take?

The CLI stays free and open. Several capabilities could sit on top — which ones are worth building? Click what you'd actually use. One tap each, anonymous.

Votes stay on your device (localStorage) and fire one anonymous pageless event per choice — no accounts, no cookies, no tracking across sites.
